This worklog covers my 12-week internship journey at AWS First Cloud Journey, starting from April 2026. Throughout the program, I explored a wide range of AWS services through hands-on labs and completed a final capstone project building a cloud-native Document Management System (DMS).

The internship spanned approximately 3 months with the following weekly highlights:

**Week 1:** [AWS account setup, AWS Credits, Budgets, and IAM fundamentals](1.1-week1/)

**Week 2:** [VPC networking — Subnets, NAT Gateway, Site-to-Site VPN](1.2-week2/)

**Week 3:** [Libreswan VPN, Hybrid DNS, VPC Peering, and Transit Gateway](1.3-week3/)

**Week 4:** [AWS Backup, SMB Storage Gateway, S3 Static Website, Cross-Region Replication](1.4-week4/)

**Week 5:** [VM Migration, Amazon FSx Multi-AZ, CloudFront CDN, S3 Versioning](1.5-week5/)

**Week 6:** [Security Hub, Lambda ChatOps (auto start/stop EC2), ABAC with IAM, KMS encryption, CloudTrail, Athena](1.6-week6/)

**Week 7:** [RDS with Node.js, Kinesis/Glue/Athena data pipeline, DynamoDB CRUD, OpenSearch + Bedrock Zero-ETL, DynamoDB Streams & Kinesis Lambda](1.7-week7/)

**Week 8:** [AWS Cost analysis with Athena, DynamoDB with Python (boto3), Airbnb Data Lake, Kinesis/EMR/Redshift analytics, QuickSight dashboards](1.8-week8/)

**Week 9:** [Final project research — DMS business requirements analysis and AWS architecture design](1.9-week9/)

**Week 10:** [DMS Frontend development, Lambda backend (upload/download, Presigned URL, DynamoDB)](1.10-week10/)

**Week 11:** [Full AWS backend deployment — CloudFront, Cognito, SQS, GuardDuty, CloudWatch, SNS](1.11-week11/)

**Week 12:** [End-to-end system testing, bug fixes, documentation, and final project submission](1.12-week12/)
